******* Инструкция по работе с приложением ConsoleFTPclient.jar *******

Предполагается, что проект собран в самостоятельный файл и отделен от среды
разработки. Хотя что так, что эдак программа при старте выдает меню
аутентификации, а затем пользовательское меню. Однако автономный запуск и
сборка его, в том числе, через консоль 'штука хитрая' см. ниже.

Согласно задания FTP клиент устанавливает связь с FTP сервером, на котором
расположен файл с информацией о студентах кафедры в виде JSON файла. При
отсутствии данного файла на сервере или при наличии файла с
недокументированной структурой JSON файла программа немедленно прекращает
работу.

Если все условия соблюдены и валидные данные сервера у вас есть, то:
1. Запустите ConsoleFTPclient.jar - файл в терминале, введя строку:

java -jar ConsoleFTPclient.jar

при успешном запуске вы попадете в меню, где нужно ввести IP, логин и
пароль доступа к FTP серверу. Если связь с сервером есть, и вы все ввели
правильно, то попадете в основное меню программы.

Если связи с сервером нет или вы ввели некорректный пароль или логин,
то программа предложит повторить ввод данных. Внимание, у вас будет три
попытки.

!!! Примечание! 'Та самая хитрость' c UTF – 8 !!!
Существует вероятность, что после запуска программы и КОРРЕКТНОГО ввода
всех данных вылетает ошибка о невозможности прочесть JSON файл, хотя он
расположен в нужном месте и имеет правильную структуру и правильное имя:
students.json. (Так же при работе программы русский шрифт будет
отображаться виде псевдосимволов)

При возникновении подобной ошибки запустите программу повторно с флагом:

java -jar -Dfile.encoding=UTF-8 ConsoleFTPclient.jar

!!! Так же возможна ошибка при работе с регулярными выражениями, а это
уже проблема при сборке программы, т.е. необходимо применить тот же флаг
при сборке !!!

2. После прохода меню валидации и при успешном соединении вы попадете в
основное меню работы с данными:

Меню доступных команд (для повторного вывода меню, наберите - HELP):
        + "list - показать список студентов отсортированных по имени;
        + "info id - показать информацию о студенте по его ID;
        + "add name - добавить студента в базу данных;
        + "remove id - удалить студента из базы данных по его ID;
        + "exit - завершить работу программы и сохранить все изменения.

Действовать согласно подсказкам меню.
3. Как только вы попадаете в меню базы данных, файл с информацией будет
   импортирован в программу. После выполнения каждой команды данные будут
   автоматически сохраняться на сервере. При разрыве связи последние
   изменения не сохранятся. Соответствующее сообщение выходит на экран.
4. При выходе из программы данные передаются на сервер автоматически.

ПРИМЕЧАНИЕ! Программа работает в двух режимах связи с FTP
сервером (активном и пассивном). При первоначальном обращении
к серверу FTP клиент работает в активном режиме, если FTP
сервер стоит за брэндмауэром, то программа переходит в
пассивный режим обмена данными, соответствующее сообщение
выводится на экран.
